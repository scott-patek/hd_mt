# Security Policy

## Reporting a Vulnerability

Please do not open a public GitHub issue for suspected security vulnerabilities.

Report privately through GitHub security reporting if it is enabled for the
repository. If private reporting is not available, contact the maintainer at
spatek01@att.net before publishing details.

Include the following when possible:

- A description of the issue and affected version
- Steps to reproduce or a proof of concept
- Any known impact or user-facing risk
- Suggested mitigations or workarounds, if known

I will review reports on a best-effort basis and coordinate a fix or disclosure path
when the report is reproducible and in scope.

## Scope

This project is a desktop audio mastering analysis application. Reports are most useful
when they involve one of the following:

- Malicious file handling
- Unsafe installer or packaging behavior
- Unexpected network behavior
- Credential or token exposure in project automation
